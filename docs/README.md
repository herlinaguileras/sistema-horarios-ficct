Documentación del Proyecto - Sistema de Horarios FICCT

Esta carpeta contiene la documentación técnica y de proyecto organizada según el modelo PUDS y UML.

Estructura principal:

- `CARATULA.md` - Portada del documento
- `TOC.md` - Tabla de contenido (actualizable antes de cada entrega)
- `perfil.md` - Perfil del proyecto (introducción, objetivos, alcance)
- `marco-teorico.md` - Marco teórico y referencias
- `modelo-negocio.md` - Modelo de negocio / Canvas
- `requerimientos/` - Actores, Casos de Uso, wireframes
- `analisis/` - Análisis de arquitectura, clases y paquetes
- `diseno/` - Diseño lógico y físico de datos, ERD, scripts SQL
- `implementacion/` - Detalles de despliegue, herramientas y comandos
- `diagrams/` - PlantUML (.puml) y exportaciones
- `anexos/` - SQL, capturas y otros anexos

Antes de cada entrega principal actualiza `TOC.md` con fecha y versión.

Guía rápida: los diagramas deben mantenerse en PlantUML (`.puml`) para versionado y exportarse a PNG para presentaciones.
